## 原理

视觉格式化模型描述元素盒子如何参与布局、定位和绘制。`display` 决定盒子的内外显示类型，`position` 决定是否脱离普通流或建立定位关系，`float` 是历史图文环绕模型，`overflow` 控制内容溢出和滚动容器，`z-index` 参与层叠顺序。

| 属性 | 解决的问题 | 关键边界 |
|---|---|---|
| `display` | 元素生成什么盒子、内部如何布局 | `block`、`inline`、`flex`、`grid`、`contents` |
| `position` | 元素如何定位 | `static`、`relative`、`absolute`、`fixed`、`sticky` |
| `float` | 图文环绕 | 不适合作为现代整体布局系统 |
| `overflow` | 溢出裁剪、滚动容器 | 会影响 BFC、sticky、滚动链路 |
| `z-index` | 同层叠上下文内的绘制顺序 | 不能跨层叠上下文比较 |

`display` 的现代写法可以拆分为外部显示和内部显示，例如 `display: block flex`；日常仍常用 `display: flex`、`grid` 等简写。

## 用法

定位示例：

```css
.tooltip-anchor {
  position: relative;
}

.tooltip {
  position: absolute;
  inset-block-end: 100%;
  inset-inline-start: 0;
  z-index: 10;
}
```

`sticky` 需要滚动容器和偏移值：

```css
.section-title {
  position: sticky;
  top: 0;
  background: Canvas;
}
```

`overflow` 不只是“显示滚动条”。它可能创建滚动容器，也可能影响 sticky 参考对象：

```css
.panel {
  max-block-size: 60vh;
  overflow: auto;
}
```

`display: contents` 会让元素本身不生成盒子，但子元素仍参与父级布局；它可能影响可访问性树和语义表达，使用时要验证。

## 实践

视觉格式化排查步骤：

1. 元素不占位：查看 `display` 是否为 `none`、`contents` 或被绝对定位。
2. 绝对定位错位：检查最近定位祖先和 transform 祖先。
3. sticky 不生效：检查祖先 `overflow`、滚动容器和 `top` 等偏移。
4. 内容被裁剪：检查 `overflow`、border radius、contain 和父级高度。
5. z-index 无效：检查是否进入新的层叠上下文。

```css
.scroll-region {
  overflow: auto;
  overscroll-behavior: contain;
}

.scroll-region__header {
  position: sticky;
  inset-block-start: 0;
  z-index: 1;
}
```

## 陷阱

| 常见误区 | 风险 | 建议 |
|---|---|---|
| 用 float 做整页布局 | 清除、等高、响应式维护困难 | 使用 flex/grid |
| sticky 父级设置 `overflow: hidden` | sticky 参考滚动容器变化或失效 | 明确滚动容器 |
| fixed 总是相对视口 | transform 祖先可能改变包含块 | 检查祖先属性 |
| `overflow: hidden` 裁剪浮层 | 下拉、阴影、focus ring 被切掉 | 浮层放到合适层级或避免裁剪 |
| `display: contents` 滥用 | 语义和可访问性可能受影响 | 关键语义元素慎用 |
