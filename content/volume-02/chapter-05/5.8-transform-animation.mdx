## 背景与问题

CSS 变换不会改变元素在文档流中的占位，它在绘制阶段对元素进行平移、旋转、缩放或倾斜。动画则是在时间轴上改变属性值。常见动画方案有三类：

| 方案 | 适合场景 | 特点 |
|---|---|---|
| `transition` | 状态之间的简单过渡 | 声明少，依赖属性变化触发 |
| `@keyframes` | 循环或多阶段动画 | 纯 CSS，可复用 |
| Web Animations API | 需要 JS 控制播放、暂停、反转、时间轴 | 适合复杂交互和状态机 |

在前端工程中，变换与动画：transform、transition、@keyframes、FLIP、Web Animations API通常不是单独存在的知识点，而是连接语言特性、浏览器能力、框架抽象和工程约束的判断入口。理解它的背景，可以帮助我们回答三个问题：为什么需要这个机制、它解决了什么边界问题，以及在什么场景下不应该过度使用。

## 工作原理

性能上要关注渲染管线。动画 `width`、`height`、`top`、`left` 等属性通常会触发布局或绘制；动画 `transform` 和 `opacity` 更容易由合成线程处理，适合高频交互。

FLIP 是一种思路：First 记录初始位置，Last 记录最终位置，Invert 用反向 transform 抵消视觉跳变，Play 再把 transform 过渡回正常状态。它常用于列表重排、卡片展开和路由切换。

## 代码示例

简单状态变化使用 `transition`：

```css
.button {
  transform: translateY(0);
  transition:
    transform 160ms ease,
    box-shadow 160ms ease;
}

.button:hover {
  /* 中文注释：使用 transform 模拟上浮，避免修改布局位置 */
  transform: translateY(-2px);
}
```

循环动画使用 `@keyframes`：

```css
@keyframes pulse {
  from {
    opacity: 0.6;
    transform: scale(0.98);
  }

  to {
    opacity: 1;
    transform: scale(1);
  }
}

.status-dot {
  animation: pulse 900ms ease-in-out infinite alternate;
}
```

需要 JS 控制时使用 Web Animations API：

```ts
const panel = document.querySelector<HTMLElement>(".panel")

if (panel) {
  const animation = panel.animate(
    [
      { opacity: 0, transform: "translateY(8px)" },
      { opacity: 1, transform: "translateY(0)" },
    ],
    {
      duration: 180,
      easing: "ease-out",
      fill: "both",
    },
  )

  // 中文注释：业务状态取消时，可以直接终止动画并回到初始状态
  animation.cancel()
}
```

尊重用户减少动态效果偏好：

```css
@media (prefers-reduced-motion: reduce) {
  .status-dot,
  .button {
    /* 中文注释：保留最终样式，减少非必要运动 */
    animation: none;
    transition: none;
  }
}
```

## 工程实践

动画设计应先回答三个问题：

1. 这个运动是否帮助用户理解状态变化，而不是纯装饰。
2. 它是否可以被取消、跳过或在减少动画偏好下禁用。
3. 它是否在低性能设备上仍然稳定。

FLIP 的核心代码结构如下：

```ts
function animateFlip(element: HTMLElement, mutateLayout: () => void) {
  const first = element.getBoundingClientRect()

  mutateLayout()

  const last = element.getBoundingClientRect()
  const deltaX = first.left - last.left
  const deltaY = first.top - last.top

  // 中文注释：先把元素视觉上放回旧位置，再过渡到新位置
  element.animate(
    [
      { transform: `translate(${deltaX}px, ${deltaY}px)` },
      { transform: "translate(0, 0)" },
    ],
    {
      duration: 220,
      easing: "cubic-bezier(.2, 0, .2, 1)",
    },
  )
}
```

调试动画时使用 Performance 面板观察是否频繁 Layout、Paint；使用 Layers 或 Rendering 工具检查合成层。不要长期给大量元素设置 `will-change`，它会占用内存，应只在即将动画前短暂启用。

## 常见误区

| 常见问题 | 处理建议 |
|---|---|
| 动画 `height: auto` | 用 grid、max-height、FLIP 或 JS 测量实际高度 |
| 用 `top/left` 做拖拽 | 改用 `transform: translate()` |
| 动画太多且不可关闭 | 对非关键动画接入 `prefers-reduced-motion` |
| 滥用 `will-change` | 只对少量即将动画的元素临时使用 |
| 动画和业务状态脱节 | JS 动画需要处理取消、完成、异常和重复触发 |
