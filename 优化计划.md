# 前端开发百科全书优化计划

> 本文件用于持续记录章节补充、审校、结构调整和发布验收状态。每次补充内容后，建议同步更新对应章节状态、最近更新日期和补充记录。

## 计划目标

- 补齐当前 MDX 骨架章节，保证每篇文章至少具备「原理、用法、实践、陷阱」四个有效内容段。
- 统一章节难度、内容状态、前置依赖、相关推荐和交叉引用，提升知识路径的连续性。
- 将单节专题章统一纳入「合并、扩展、保留」决策，避免目录层级碎片化。
- 为后续分批补充内容提供可追踪的状态表和验收标准。

## 状态说明

| 状态 | 含义 | 进入下一状态的条件 |
|---|---|---|
| 骨架 | 只有标准标题或占位内容，正文尚未展开 | 完成至少 1 篇小节正文 |
| 补写中 | 已开始补充，仍存在明显空段或未覆盖小节 | 全章小节均有有效正文 |
| 待审校 | 正文基本完成，等待术语、链接、代码和示例检查 | 完成审校并通过构建 |
| 基本完成 | 已有完整正文，但仍可继续补充案例、图示或交叉引用 | 完成最终审校 |
| 已完成 | 内容、结构、交叉引用、构建验证均通过 | 持续维护 |

## 优先级说明

| 优先级 | 说明 |
|---|---|
| P0 | 影响主学习路径或当前内容质量断层，优先补齐 |
| P1 | 重要主题，建议在基础链路完成后补齐 |
| P2 | 专题扩展或高级内容，可按需求逐步补充 |

## 单篇文章完成标准

| 检查项 | 标准 |
|---|---|
| Frontmatter | `title`、`description`、`volume`、`chapter`、`section`、`tags`、`difficulty`、`prerequisites`、`related`、`crossRefs`、`lastUpdated` 完整 |
| 正文结构 | 至少包含「原理、用法、实践、陷阱」四个二级标题，且每段都有有效正文 |
| 原理 | 解释概念背景、核心机制、关键术语和设计取舍 |
| 用法 | 提供 API、语法、配置或操作步骤，包含最小可运行示例 |
| 实践 | 至少给出 1 个真实业务场景或工程落地案例 |
| 陷阱 | 提供常见错误、表现、原因和解决方案 |
| 组件 | 需要时使用 `Callout`、`SectionCard`、`Quiz` 等 MDX 组件增强阅读 |
| 关联 | 补充前置知识、后续延伸、相关章节和交叉引用 |
| 验证 | 生成搜索索引并通过静态构建 |

## 章节完成标准

- 章内所有小节均达到单篇文章完成标准。
- 章节内难度递进合理，不把高级内容标记为基础。
- 章节内小节顺序符合学习路径，必要时调整目录顺序。
- 章内至少建立 3 条有效相关章节或交叉引用。
- 章节补充完成后更新本文件的状态、最近更新日期和补充记录。
